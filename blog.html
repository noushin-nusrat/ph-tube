Let's talk about Var, Let, and Const.

Var

Var is JavaScript variable declaration Keyword. Var used to very popular before ES6, infect there wasn't any other declaration Keyword. There are issues associated with variables declared with var, though. That's why it was necessary to update some Keyword and thankfully JavaScript developers came with a solution that is update JavaScript language. Var variables array hoisted, and behave like global scope.

Let

Let is very powerful Keyword of JavaScript. It also solves the problem with var that we just covered. Let is a block scoop variable, and values are changeable. you can update and change the value whenever you want inside the scoop. But it can not be re-declare.

Const

Variables declared with const maintain constant values. Const declarations share some similarities with let declarations. But const values can not be update or re-declared. const also block scoop variable

What is null?

We have heard the word "null" many times in English. Null in JavaScript means an empty value and is also a primitive type in JavaScript. Null is basically an assignment value given to a variable.

What is Undefine?

As the name suggests, undefined means "not defined". So we declare a variable but do not assign a value to it, the variable becomes undefined.The variable gets created at the run-time. When we do not pass an argument for a function parameter, the default value is taken as undefined. Besides, when a function doesn't return a value, it returns undefined.

What is REST API?

A RESTful API is an architectural style for an application program interface (API) that uses HTTP requests to access and use data. That data can be used to GET, PUT, POST and DELETE data types, which refers to the reading, updating, creating and deleting of operations concerning resources.
